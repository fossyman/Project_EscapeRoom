<h2>Chapter 2: Project Layout</h2>
Below are the most important files

- __Assets__
This folder contains all of the assets for the project (Please keep assets in their named locations)
- __RAW_ASSETS__
This is where all of the raw assets will go. These include:
	- Blender's __.BLEND__ files.
	- Photoshop's __.PSD__ files.
	- Krita's __.KRA__ files.
	- Audacity's __.AUP3__ files.

- __Scenes__
This is where all of the scenes will be placed.
Remember that scenes are the container for everything, like levels, characters, and props.

- __Src__
This is the location for all of the code-related assets. This will be .gd, .cs

<h2>Chapter 3: Asset Workflow</h2>

__3D__
__Creating new asset__
Raw project files (.blend) are to be placed in their respective folder in the RAW_ASSETS folder.
Exported assets is to be placed in their respective folder in the Assets folder
__Modifying new asset__
Modifications should be done to the file in the RAW_ASSETS folder and exported to the same location in the Assets folder.

__Exporting Assets__
3D assets are __required to be exported into the .glTF 2.0 format__
Alternative versions of the assets can be saved as *-v2

<br><br>

__2D__
__Creating new asset__
Raw project files (.kra, .psd) are to be placed in their respective folder in the RAW_ASSETS folder.
Exported assets is to be placed in their respective folder in the Assets folder
__Modifying new asset__
Modifications should be done to the file in the RAW_ASSETS folder and exported to the same location in the Assets folder.
Alternative versions of the assets can be saved as *-v2

__Exporting Assets__
Keep 2D assets as either .PNG, or .JPG. __GIFS WILL NOT WORK__

<br><br>

__Audio__
__Creating new asset__
Raw project files (.AUP) are to be placed in their respective folder in the RAW_ASSETS folder.
Exported assets is to be placed in their respective folder in the Assets folder
__Modifying new asset__
Modifications should be done to the file in the RAW_ASSETS folder and exported to the same location in the Assets folder.
Alternative versions of the assets can be saved as *-v2

__Exporting Assets__
Keep audio assets as either .WAV, or .OGG OR .MP3.
WAV: biggest file size, best quality. Use for important audio, like music.
MP3: smaller file size, okay quality. Use for not so important assets, like sound effects.
OGG: smallest file size, least quality. use for the least important audio assets.

<br><br>
